# RULES.md — 개발 규칙

## 구조 원칙

- 툴은 `tools/` 폴더 안에 독립 디렉토리로 추가한다.
- 툴 하나를 추가할 때 `base/` 코드를 수정하지 않는 것을 목표로 한다.
- 툴끼리 서로 import하지 않는다. (의존성 교차 금지)

## 베이스 수정 원칙

- 2개 이상의 툴에서 같은 코드가 반복될 때만 `base/utils`에 올린다.
- "나중에 쓸 것 같아서" 미리 만들지 않는다.

## 설정 원칙

- 전역 설정은 루트 `config.yaml`에만 둔다.
- 툴 전용 설정은 해당 툴 폴더 안에 둔다.
- 설정 파일에 시크릿(패스워드, API 키)을 직접 쓰지 않는다.
  → `.env` 파일이나 환경변수로 관리.

## 로그 원칙

- 모든 로그는 `logs/` 폴더에 쌓인다.
- 툴 이름을 로그에 항상 태깅한다.
- `print()`로 디버깅하고 로그에 안 남기는 것 금지.

## 파일/네이밍 원칙

- 폴더명: 소문자 하이픈 구분 (예: `tool-name`)
- 파일명: 소문자 언더스코어 구분 (예: `main.py`, `file_handler.py`)
- 툴 폴더마다 `README.md` 필수 (한 줄이라도 설명 있어야 함)

## 하지 말아야 할 것들

- 툴 하나에 너무 많은 기능 욱여넣기
- 쓰지도 않을 추상화 계층 미리 만들기
- 베이스를 툴 전용 기능으로 오염시키기
- 로그 없이 조용히 실패하게 두기
