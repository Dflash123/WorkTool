# Monster Skill Analyzer — 팀 사용 가이드

> **이 문서는 Notion 또는 Google Docs에 그대로 붙여넣기 하거나 임포트해서 사용할 수 있습니다.**
> Notion: 새 페이지 → `/` 입력 → Import → Markdown 선택

---

## 이 툴은 무엇인가요?

게임 몬스터 스킬의 타이밍, 가이드 이펙트, 플레이어 체감을 **구조적으로 기록하고 분석**하는 기획팀 전용 도구입니다.

테스트 직후 빠르게 데이터를 입력하면, 툴이 자동으로 공정성 지표를 계산하고 위험 패턴을 탐지합니다.

모든 데이터는 **팀 공유 Google Sheets**에 저장되어 누구나 실시간으로 확인할 수 있습니다.

---

## 처음 설치하기 (최초 1회)

### 1단계 — Python 설치

1. [python.org](https://www.python.org/downloads/) 접속
2. Python 3.10 이상 다운로드
3. 설치 시 **"Add Python to PATH"** 반드시 체크

### 2단계 — 툴 폴더 받기

팀장에게 `monster-skill-analyzer` 폴더를 받아 원하는 위치에 복사합니다.

### 3단계 — Google 인증 파일 배치

팀장에게 `service_account.json` 파일을 받아 다음 위치에 넣습니다.

```
monster-skill-analyzer/
└── credentials/
    └── service_account.json   ← 여기에 넣기
```

### 4단계 — 설정 입력

`src/config/config.yaml` 파일을 메모장으로 열어 아래 항목을 수정합니다.

```yaml
google_sheets:
  credentials_file: "credentials/service_account.json"
  spreadsheet_id: "팀장에게 받은 ID를 여기에 붙여넣기"
```

> Spreadsheet ID는 구글 시트 URL에서 확인할 수 있습니다.
> `https://docs.google.com/spreadsheets/d/【이 부분이 ID】/edit`

### 5단계 — 실행

`run.bat` 파일을 더블클릭합니다.

- 처음 실행 시 패키지 자동 설치 (인터넷 필요, 약 2~3분)
- 이후 실행은 브라우저가 바로 열립니다

## 매일 사용하는 방법

```
run.bat 더블클릭 → 브라우저 자동 오픈 → 사용
```

## 각 메뉴 상세 가이드

### 📝 스킬 입력

스킬의 타이밍 수치와 가이드 이펙트 정보를 입력합니다.

**타이밍 입력 항목**

| 항목 | 설명 | 단위 |
|------|------|------|
| 선딜 | 공격 모션 시작 ~ 판정 발생까지 | ms |
| 판정 지속 | 실제 피해 판정이 활성화된 시간 | ms |
| 후딜 | 판정 종료 ~ 다음 행동 가능까지 | ms |
| 회피 가능 시작 | 플레이어가 회피 입력을 시작해야 하는 시점 | ms |
| 회피 가능 종료 | 회피 입력이 유효한 마지막 시점 | ms |

> **총 행동 시간**과 **실제 반응 가능 시간**은 자동 계산됩니다.

**입력 예시**

```
선딜: 800ms  판정: 400ms  후딜: 600ms
회피 시작: 300ms  회피 종료: 700ms
→ 총 행동: 1800ms  반응 가능: 400ms
```

**가이드 이펙트 항목**

| 항목 | 설명 |
|------|------|
| 가이드 존재 여부 | 토글로 ON/OFF |
| 가이드 시작 시점 | 가이드가 화면에 표시되기 시작하는 시점 (ms) |
| 가이드 유형 | visual / sound / both |
| 가이드 강도 | 1(매우 약함) ~ 5(매우 강함) |
| 판정과 일치 여부 | 가이드 범위가 실제 판정 범위와 일치하는지 |

---

### 🧪 테스트 로그

테스트를 진행할 때마다 기록합니다. **체감 평가와 연결**하기 위해 반드시 먼저 생성하세요.

**입력 항목**

- 대상 스킬 선택
- 테스트 유형: `internal` (내부) / `live` (라이브)
- 테스트 날짜, 담당자 이름, 버전, 환경
- 참여 인원, 메모

---

### ⭐ 체감 평가

테스트 참가자가 직접 입력하거나, 담당자가 대신 기록합니다.

**점수 항목 (1~5점)**

| 항목 | 1점 | 5점 |
|------|-----|-----|
| 반응 시간 충분성 | 전혀 시간이 없었다 | 충분히 여유로웠다 |
| 피격 납득도 | 완전히 억울했다 | 납득이 됐다 |
| 가이드 명확성 | 가이드가 없거나 불명확 | 매우 명확했다 |
| 공격 가독성 | 공격 모션을 전혀 읽을 수 없었다 | 즉시 파악됐다 |
| 학습 가능성 | 반복해도 피할 수 없을 것 같다 | 익히면 피할 수 있을 것 같다 |
| 스트레스 | 스트레스 없었다 | 매우 스트레스받았다 |
| 재도전 의사 | 다시 하고 싶지 않다 | 다시 도전하고 싶다 |

> **스트레스**는 낮을수록 좋은 점수입니다.

### 📊 분석 대시보드

#### 자동 계산 지표

| 지표 | 의미 | 위험 기준 |
|------|------|-----------|
| 반응 여유 지수 | 회피 가능 시간의 여유 (500ms = 만점) | 50 미만 → 경고 |
| 타이밍 공정성 | 반응여유 + 가이드품질 + 판정일치 종합 | 40 미만 → 위험 |
| 억까 가능성 | 억울한 피격 발생 가능성 | 60 초과 → 즉시 검토 |
| 설계 의도 적중률 | 기획 의도대로 체감하는지 | 70 미만 → 재검토 |
| 학습 가능성 | 패턴 학습 가능 여부 | 60 미만 → 경고 |

#### 위험도 등급

| 등급 | 의미 | 대응 |
|------|------|------|
| 🟢 LOW | 정상 | 현행 유지 |
| 🟡 MEDIUM | 주의 | 다음 버전에서 개선 검토 |
| 🔴 HIGH | 즉시 검토 | 수정 후 재테스트 필수 |

#### 위험 패턴 자동 탐지

다음 조건이 발견되면 자동으로 경고를 표시합니다.

- 반응 가능 시간이 250ms 미만 (인간 반응속도 이하)
- 가이드 없는 스킬의 억까 가능성 60 초과
- 판정 범위와 가이드 범위 불일치
- 스트레스 평균 4 초과 + 재도전 의사 평균 2 미만

## 자주 묻는 질문

**Q. 앱이 켜지지 않아요.**

Python이 설치되어 있는지 확인하세요. 설치 시 "Add Python to PATH"를 체크했는지 확인하세요.

**Q. 실행은 되는데 데이터가 저장/불러와지지 않아요.**

`⚙️ 설정` 메뉴에서 Spreadsheet ID와 인증 파일 경로를 확인하고 연결 테스트를 눌러보세요.

**Q. 팀원이 동시에 입력해도 되나요?**

가능합니다. 다만 같은 스킬을 동시에 수정하는 경우 마지막 저장이 우선됩니다.

**Q. Google Sheets 계정을 바꾸고 싶어요.**

`⚙️ 설정` 메뉴에서 Spreadsheet ID와 인증 파일명을 변경하고 저장하면 됩니다.

**Q. 데이터를 엑셀로 내보내고 싶어요.**

분석 대시보드 → 전체 요약 탭 → **CSV 내보내기** 버튼을 클릭하세요.

## 팀장/관리자 가이드

### Google Sheets 신규 설정 방법

1. [Google Cloud Console](https://console.cloud.google.com/) 접속
2. 새 프로젝트 생성
3. **API 및 서비스 → 라이브러리** → `Google Sheets API`, `Google Drive API` 각각 사용 설정
4. **사용자 인증 정보 → 서비스 계정 만들기**
5. 서비스 계정 → **키 → 새 키 추가 → JSON** → 다운로드
6. 새 Google 스프레드시트 생성
7. 스프레드시트 **공유** → 서비스 계정 이메일 추가 (편집자 권한)
   - 이메일: `service_account.json` 안의 `client_email` 값
8. 스프레드시트 URL에서 ID 복사

### 팀원에게 배포 시 전달할 파일

- `monster-skill-analyzer/` 폴더 전체
- `service_account.json` (보안 채널로 별도 전달)
- Spreadsheet ID (메시지로 공유)

### 시트 탭 구조 (자동 생성됨)

| 탭 이름 | 내용 |
|---------|------|
| SkillDefinitions | 스킬 정의 원본 데이터 |
| TestLogs | 테스트 회차 이력 |
| PlayerFeedback | 체감 평가 원본 데이터 |
| BalanceHistory | 밸런스 수정 이력 |

> 첫 실행 시 탭이 없으면 자동으로 생성됩니다.
