# 자동 계산 지표 정의

모든 지표는 `src/modules/metrics_calculator.py`에서 계산.
입력값 변경 시 실시간 재계산.

---

## 1. 반응 여유 지수 (Reaction Margin Index)

**의미**: 플레이어가 회피를 시도할 수 있는 시간적 여유

```
reaction_window = dodge_end_ms - dodge_start_ms

반응여유지수 = min(reaction_window / 500, 1.0) * 100
```

- 기준: 500ms 이상이면 만점 100
- 인간 평균 반응속도 250ms 기준으로 500ms = 여유 2배
- 범위: 0~100
- 위험 기준: 50 미만 → 경고 표시

---

## 2. 타이밍 공정성 점수 (Timing Fairness Score)

**의미**: 해당 스킬이 얼마나 "공정하게" 설계되었는지 종합 점수

```
guide_score = guide_intensity / 5 * 100  (가이드 없으면 0)
match_bonus = 20 if guide_match else 0

타이밍공정성 = (반응여유지수 * 0.5) + (guide_score * 0.3) + match_bonus
```

- 범위: 0~100
- 위험 기준: 40 미만 → 재설계 권고

---

## 3. 억까 가능성 지수 (Unfair Hit Index)

**의미**: 플레이어 입장에서 "억울한 피격"이 발생할 가능성

```
억까가능성 = 100 - 타이밍공정성점수
```

- 범위: 0~100
- 위험 기준: 60 초과 → 즉시 검토 필요

---

## 4. 설계 의도 적중률 (Design Intent Hit Rate)

**의미**: 기획 의도대로 플레이어가 체감하고 있는지 비율

```
체감평균 = (반응충분성 + 피격납득도 + 가이드명확성 + 학습가능성) / 4
설계의도적중률 = 체감평균 / 5 * 100
```

- 범위: 0~100
- 기준: 70 이상이면 의도 달성

---

## 5. 학습 가능성 점수 (Learnability Score)

**의미**: 반복 플레이 시 플레이어가 패턴을 학습할 수 있는 정도

```
학습가능성점수 = (learnability * 0.6 + guide_clarity * 0.4) / 5 * 100
```

- 범위: 0~100
- 기준: 60 이상이면 학습 가능

---

## 지표 요약 테이블

| 지표 | 범위 | 위험 기준 | 색상 |
|------|------|-----------|------|
| 반응여유지수 | 0~100 | < 50 | 빨강 |
| 타이밍공정성 | 0~100 | < 40 | 빨강 |
| 억까가능성 | 0~100 | > 60 | 빨강 |
| 설계의도적중률 | 0~100 | < 70 | 노랑 |
| 학습가능성 | 0~100 | < 60 | 노랑 |

---

## 위험 패턴 자동 탐지 규칙

| 조건 | 경고 메시지 |
|------|------------|
| reaction_window < 250ms | 인간 반응속도 미만 - 회피 불가능 구간 |
| guide_exists = False AND 억까가능성 > 60 | 가이드 없는 고위험 스킬 |
| guide_match = False | 판정 범위와 가이드 불일치 |
| 설계의도적중률 < 50 | 기획 의도 미달성 - 재설계 검토 |
| stress > 4 AND retry_intent < 2 | 스트레스 과다 - 이탈 위험 |
